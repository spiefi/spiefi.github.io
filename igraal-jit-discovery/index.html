iGraal Smart Discovery

01 — Problem

The Discovery Gap

iGraal has 10M+ users in France and a vast merchant catalogue. But the current experience is passive — users must know what they want, search for a specific merchant, and hope there's a deal. The app doesn't meet users where they are.

🔴 The User Problem

"I want to save money, but I don't want to work for it."

Users have varying levels of intent — from "I need Nike shoes" to "surprise me with something good." The current merchant-list model serves only high-intent users who already know the brand. Everyone else bounces.

🟡 The Business Problem

High-value offers go unseen because discovery is merchant-indexed, not need-indexed.

A user searching for "shoes" won't find a 12% cashback on Zalando unless they already know Zalando is a partner. The platform leaves money on the table for both users and merchants.

🟢 The Opportunity

Move from merchant discovery → intent discovery. Let the app understand what users want and compose the answer in real time.

Three interaction modes. One JIT-composed output. Every user type — from askers to browsers — gets a relevant, personalized deal surface.

Core insight: Competitors (Honey, Rakuten, ShopBack) are all merchant-indexed. No one has cracked intent-indexed discovery with AI-composed screens yet. iGraal can be first in Europe.

03 — Interaction Modes

Three Paths, One Destination

Every mode leads to the same JIT-composed answer screen. The entry differs, the output is unified. Users don't choose a mode — the app detects it from their behavior.

🗣️
Ask Mode
Conversational — natural language → AI-composed answer
  1. Entry: User taps the Ask prompt or speaks a question. "Best deal on headphones under €80"
  2. Understanding: AI parses intent → category (electronics), product type (headphones), constraint (< €80), goal (best deal)
  3. JIT Assembly: System composes a screen: top cashback merchants for headphones, active promo codes, price comparison card, savings calculator
  4. Refinement: User can follow up: "What about wireless?" → screen updates in place. No page reload.
  5. Action: User taps "Go to store" → activates cashback → redirected to merchant with tracking
Personas: Chloé, Sophie
🔍
Search Mode
Structured — keyword → smart filtered results → JIT answer
  1. Entry: User taps search bar, types keyword: "Nike" or "electronics" or "travel deals"
  2. Smart Categorization: Results auto-grouped: merchants, active codes, cashback rates, trending deals — not just a flat list
  3. JIT Assembly: System composes structured result screen with filters (category, cashback %, expiry), sort options, highlight cards
  4. Progressive Enhancement: AI suggests: "Looking for Nike? Zalando has 8% cashback + free shipping code" — a nudge, not a takeover
  5. Action: Quick-tap to activate cashback. One step, not three.
Personas: Thomas, Sophie
🎁
Browse Mode
Proactive — curated feed → personalized deal surfaces
  1. Entry: User opens app with no specific intent. Lands on personalized home feed.
  2. Smart Feed: AI-curated cards based on: purchase history, browsing patterns, seasonality, trending deals, location
  3. JIT Assembly: Each feed card expands into a composed deal surface. "Your weekend travel deals" → flights, hotels, car rental — all with stacked savings
  4. Proactive Nudges: "You bought from Sephora last month — they just increased cashback to 10%" — contextual, timely, relevant
  5. Action: Swipe to save, tap to shop. Low friction, high discovery.
Personas: Léa, Marc

Design principle: The three modes aren't tabs or settings. They're behavioral states the app detects. A user who opens the app and immediately starts typing → Search Mode. A user who says "find me..." → Ask Mode. A user who just scrolls → Browse Mode. Seamless transition between modes, no friction.

05 — JIT Component Engine

How Screens Are Assembled

The JIT engine selects from a palette of composable components based on the intent analysis. No two answer screens need to look the same — they're built from the same blocks but assembled per query.

💰 Cashback Merchant Card
Shows merchant logo, cashback %, status (active/pending), and "Shop Now" CTA. Primary building block.
data: merchant, rate, status, logo, deeplink
🎟️ Promo Code Card
Code with copy button, validity date, success rate badge ("92% success"), and conditions.
data: code, merchant, expiry, successRate, conditions
📊 Savings Comparison
Side-by-side comparison of 2-3 merchants for the same product category. Shows total savings breakdown.
data: merchants[], rates[], estimatedSavings[]
🧮 Stacked Savings Calculator
For power users. Shows cashback + promo code + voucher = total savings. Interactive amount input.
data: cashback, promoValue, voucherValue, total
⏰ Urgency Banner
Countdown for expiring deals. "This offer ends in 4h 23m." Drives FOMO without being manipulative.
data: expiryTimestamp, dealTitle, merchant
📈 Cashback Trend Graph
Shows cashback rate history for a merchant. "Rate is at 12-month high" or "Usually higher in December."
data: merchant, rateHistory[], currentRate, trend
🎁 Deal Bundle Card
Groups related offers: "Your complete travel package: flights + hotel + car, total cashback: €47"
data: bundleTitle, offers[], totalSavings
🔔 Alert Setup Card
When no current deal exists: "Set an alert for Nike cashback above 8%." Turns no-result into retention.
data: category, threshold, frequency, channel
💬 Clarification Chip Row
Quick-tap refinement options: "Under €50" / "Free shipping" / "Top-rated" / "Ending soon". Replaces typing.
data: options[], selectedFilters[], queryContext

Assembly logic: The Intent Engine scores each component for relevance (0-1) based on the parsed query. Components above 0.6 threshold are included. Order is determined by: relevance score × user preference weight × recency. The result is a VStack of components — familiar, scrollable, native-feeling.

06 — Proactive Layer

Don't Wait to Be Asked

The most powerful feature isn't what happens when users ask — it's what the app offers before they ask. These triggers drive engagement, reactivation, and delight.

🔁 Purchase Pattern Recognition High Impact
Detect recurring purchases (monthly subscriptions, seasonal buys) and surface deals proactively.
"You usually order contact lenses around this time — Optical Center has 7% cashback right now."
📈 Cashback Rate Spike Alert High Impact
When a merchant the user has shopped at increases their cashback rate significantly.
"Fnac just doubled their cashback to 8% — your average order there would save you €14."
🎯 Contextual Moment Detection Medium Impact
Season, events, holidays, weather → surface relevant categories automatically.
"Planning your summer vacation? Here are travel deals with up to 12% cashback."
💤 Dormant User Reactivation Critical for Growth
For users who haven't opened the app in 30+ days. Show instant value on return.
"Welcome back, Marc! You have €3.40 pending + here are 3 deals for brands you love."
🏆 Savings Milestone Retention
Celebrate when users hit cashback milestones. Gamification without being cheesy.
"You've saved €100 with iGraal! Here's a bonus: exclusive 15% code for your next purchase."
👥 Social Proof Nudge Retention
Show trending activity from the iGraal community to drive urgency and discovery.
"2,400 people activated Zalando cashback today — sale ends tonight."
07 — Edge Cases & Error States

The Stuff That Kills You

Every edge case that isn't designed for becomes a production bug, a support ticket, or a churned user. These are the scenarios we must handle.

Scenario User Impact Design Response Severity
🤷 Zero results "Nothing found" kills engagement instantly Never show empty. Offer: related categories, trending deals, alert setup. "No exact match, but here's what's close." High
🌀 Vague query "Something nice" — AI doesn't know what to show Show clarification chips: category suggestions, trending picks. One guided question max before showing *something*. High
⏰ Expired deal surfaced User clicks → deal is dead. Trust destroyed. Real-time validation before display. If expired: "This deal ended 2h ago. Here's the next best option." Never show dead deals without labeling. High
🐌 Slow AI response 3+ second wait = user bounces Skeleton loading with progressive reveal. Show cached/fast results first, enrich with AI layer. "Building your personalized results..." High
🌍 Geo-restricted offers User sees deal but can't use it in their country Filter by user locale before display. If shown: clear geo badge. "Available in France only." Never let them click through to a dead end. Medium
🆕 First-time user No history = no personalization = generic experience Onboarding: "Pick 3 categories you care about." Then: trending + editorial picks until we learn. Progressive personalization — never cold. Medium
📱 Offline / poor connection AI features require network Cache last-viewed deals. Show "last updated 2h ago" with refresh option. Search/browse work offline from cache. Ask mode shows "reconnect for smart results." Medium
🔀 Mode switching mid-flow User asks a question while browsing feed Seamless transition. Context preserved. Ask bar always visible. Browsing → typing → Ask mode, no page change. Return to feed with scroll position saved. Medium
🗣️ Unparseable query "asdf" or emoji-only or language AI doesn't handle Graceful fallback: show trending deals + "Try asking something like: 'Best deal on electronics'" with example chips. Low
♿ Accessibility JIT-composed screens must be screen-reader friendly All JIT components must have semantic structure, aria labels, and logical tab order. Dynamic assembly must produce valid DOM hierarchy. Medium
🔄 Rapid re-queries User types 5 queries in 10 seconds Debounce input (300ms). Cancel in-flight requests. Latest query wins. No stale results overtaking current. Low

09 — Competitive Landscape

Why This Wins

No major cashback platform has cracked AI-driven deal discovery yet. They're all still merchant-indexed directories with search bolted on.

🍯 Honey (PayPal)

Browser extension model. Auto-applies codes at checkout. No in-app discovery. No conversation. Reactive only — you must already be on a site. Recent trust issues after affiliate fraud scandal.

No AI discovery Extension-only

🛍️ Rakuten

Merchant directory + cashback activation. Good rates, strong in US/Japan. But discovery is browse-a-list. No intent understanding. No personalized feed. Traditional e-commerce UX.

Directory model No personalization

🛒 ShopBack

Strong in SEA. Has quests (gamification), ShopBack Pay, GO (offline). Most innovative competitor. But still browse-first, no conversational discovery, no JIT composition.

Quests = good Still no AI discovery

iGraal's unfair advantage: 10M users in France. Deep merchant data. Atolls operates across FR/DE/ES/PL. Combined with JIT AI discovery → first mover in European AI-powered cashback. The window is open — but not for long. ChatGPT's Agentic Commerce Protocol is coming for shopping.

10 — Next Steps

From Here

This UX definition is the foundation. Here's what needs to happen next, in order.

1
Stakeholder Review & Alignment
Present this document to Atolls stakeholders. Validate personas, get buy-in on the 3-mode approach. Key question: which mode to build first? (Recommendation: Search Mode — highest user volume, lowest risk.)
2
JIT Component Palette Design
Define the exact visual spec for each JIT component. Size variants, data bindings, loading states, error states. This is the design system work — shared language between design and engineering.
3
Intent Engine Specification
Work with engineering to define: how does the AI parse intent? What model? What taxonomy of intents? How fast must it respond? What's the fallback when AI is down? This is the bridge between UX vision and technical reality.
4
Prototype & Test
Build a clickable prototype of the Search Mode flow with 3-4 JIT screen compositions. Test with 5-8 iGraal users. Validate: do they understand it? Is it faster than current? Does it feel trustworthy?
5
Phase 2: Ask Mode + Browse Mode
Once Search Mode validates the JIT pattern, layer on Ask Mode (conversational) and Browse Mode (proactive feed). Same component palette, different entry points. Incremental rollout, not big bang.
