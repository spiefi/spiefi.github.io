Card Inventory & Team Ownership

13 card types across 4 teams. Cards 3 and 10 are both hero banners — owned by different teams. Cards 2 and 5 are both merchant rows — different teams. The duplication is the argument: if two teams build the same card, that card belongs in the system, not in a team repo.

The Core Argument

Cross-team usage = system-level component

The designer comments (from Iza) flagged cards where multiple teams use the same pattern. That's not a coincidence — it's evidence. When CO and CCE both build a hero banner independently, you don't have two different components. You have one component built twice, with drift. The fix is one shared component in Nalu.

Section 02

Composition Model

The 9 real cards aren't 9 components. They're 3 layout templates with configurable slots. Within each family, variants differ by which slots are active — not by layout. This is the core engineering insight.

Key Insight

Commerce is not a separate card type. A commerce card in a grid = Grid Tile + price slot active. A commerce card in a list = List Row + price slot active. Don't build a separate commerce component — build price as a composable slot.

Slot Inventory

Fixed Slots (always present)
ImageVaries in ratio/position — 16:9, 1:1, 2:3, 3:2
Title1–2 lines, always visible
Variable Slots (optional, composable)
Description0–3 lines
MetaRating, date, location, category — repeatable
PriceCommerce contexts — activates the commerce mode
Badge / PillDiscount %, "new", "sponsored" — overlay or inline
CTAButton or link — optional, size varies by family
Action IconsHeart, save, compare — 0 to many
Brand ChipsMerchant logo row — hero cards only

Component Tree — Target Architecture

// One Card primitive, 3 layout variants, composable slots
<Card variant="hero" | "tile" | "row">
  ├── <Card.Image ratio="16:9" | "1:1" | "2:3" | "3:2" />
  ├── <Card.Content>
  │   ├── <Card.Title />           — always present
  │   ├── <Card.Description />     — optional
  │   ├── <Card.Meta />            — optional, repeatable
  │   └── <Card.BrandChips />     — optional, hero only
  ├── <Card.Commerce>            — optional slot group
  │   ├── <Card.Price />
  │   ├── <Card.Merchant />
  │   └── <Card.Discount />
  └── <Card.Actions>             — optional
      ├── <Card.CTA />
      └── <Card.QuickActions />   — heart, save, compare

PromoCard — New Build

The only genuinely new component. Covers full-width hero banners and image-dominant brand promo cards. Nalu has zero answer for background-image cards today.

#1 #11

  • ●layout — "hero" | "portrait" — hero is full-width landscape, portrait is image-dominant vertical

  • ✓Works without backgroundImage (solid color fallback)
  • ✓Passes WCAG AA contrast on text overlay
  • ✓Storybook stories for hero + portrait, with/without CTA, with/without badge
