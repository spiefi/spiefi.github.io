Excluded from card system
Cards 6 · 7 · 13
6
Filter / Facet Item
S&N team
A filter row is an input control, not a content container. Cards present information and navigate somewhere. Filter toggles modify a query — completely different interaction model, different accessibility semantics (role="checkbox" vs role="button"), different states.
⚠️ Mixing form controls into the card system would let filter behaviour leak into card components. Keeps a hard boundary between input and display.
7
Checkout / Order Summary
CCE team
Cards are reusable units — you put 12 in a grid. The checkout summary appears once, in a fixed layout, with business logic baked in (line items → subtotal → total → CTA). Building it as a card couples the design system to a single screen's domain logic.
⚠️ This is a page-section pattern, not a component. It belongs to the checkout flow, not the component library.
13
Empty State
All teams
Nalu already has an EmptyState component that covers this exactly — illustration, heading, body, CTA. Building it again as a card would mean two components doing the same job.
✅ Already done. Use EmptyState. No action needed.
