Section 02

Structural Grouping

The 9 real card components collapse to 4 structural patterns based on shared layout logic — not content. What varies within a pattern is only which slots are active.

Key Insight

Commerce is not a separate pattern. A commerce card in a grid = Grid Tile + price slot. A commerce card in a list = List Row + price slot. Don't build a separate commerce component — build price as a composable slot on the existing patterns.

Hero Banner
Cards 1 · 12
Full-width, image-dominant. Text overlaid or directly below. Strong CTA. Width always equals container.
Screens: Home · PDP inline
Grid Tile
Cards 3 · 9 · 10 · 11
Fixed-ratio image card. Title below or overlaid. Minimal text. Designed for visual scanning in grids.
Screens: Home · Browse · Category · Collection
List Row
Cards 2 · 4 · 5
Horizontal: image/logo left, text right, price/CTA far-right. Three density levels. The system workhorse.
Screens: Search · Sidebar · Browse lists

Section 03

Component Plan

3 base components + 1 primitive covers all 9 real card types. Every card is a variant controlled by props and active slots — not separate components.

// Target architecture
BaseCard ← primitive (keep)
├── VerticalCard ← merge VBC + VBCLite covers cards 3, 9, 10, 11
├── HorizontalCard← evolve HBC, kill OBC covers cards 2, 4, 5
└── PromoCard ← NEW covers cards 1, 12
VerticalCard

VerticalBaseCard + VerticalBaseCardLite

Merge

80% structural overlap between the two existing vertical cards. The only differences are 3 optional props. Not two components — one, with configurable slots.

Voucher Offer Card
bannerRatio=4:3 · retailerLogo · cashback · cta · expiry
Card 9

bannerRatio=1:1 · badge · cashbackPercent · noCTA
Card 10

bannerRatio=1:1 · price · saleBadge · rating
Card 11
Editorial Card
bannerRatio=16:9 · text body · author · date

Required changes

  • bannerRatioMake configurable — the 2.42:1 hardlock blocks all reuse. Add: 1:1 · 4:5 · 4:3 · 16:9 · 2.42:1 (keep as default)
  • ctaMake optional — VerticalBaseCardLite has no CTA, so the merged component must allow none
  • textAdd rich body text slot (from Lite) — needed for editorial cards
  • overlayTitleAdd title-on-image overlay mode — needed for category tiles (#3)
HorizontalCard

Evolve HorizontalBaseCard · Kill OfferBaseCard

Evolve

HorizontalBaseCard already exists but has 0 consumers. OfferBaseCard is already deprecated. Formalize the rename, make CTA optional, add compact density mode.

PromoCard

New component — the only net-new build

New

Nalu has zero answer for background-image cards with overlay content. This is the only genuinely new component in the plan. Covers two promotional contexts via a layout prop.

Card 1
Hero Promo
layout=hero · backgroundImage · overlayTitle · desc · cta · badge
Card 12
Inline Promo
layout=inline · icon · title · desc · cta · bgColor

Props to build

  • layout"hero" | "inline" — hero is full-width image-dominant, inline is compact horizontal banner
  • backgroundImageURL for background image support
  • backgroundGradientGradient overlay for text contrast on images
  • iconIcon/illustration slot — used in inline variant
BaseCard

Raw surface primitive — keep as-is

Keep

Handles appearance, border, padding, onClick. Used for brand logo tiles via composition (BaseCard + RetailerLogo). No changes needed — it's already the right abstraction.
